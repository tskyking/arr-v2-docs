## GUI demo month-puck QA

What looks good:

- `MonthPuckRail` is wired into `DashboardPage` under the ARR Movements card.
- Selected period persists independently of hover.
- Chart hover and chart bar click feed the same selected/hovered month state as the puck rail.
- Month pucks include keyboard handlers for ArrowLeft/ArrowRight/Home/End via bubbling from focused puck buttons.
- Build/typecheck passed, so the component is deployable from a frontend compilation perspective.

Risks / follow-up:

- `aria-selected` currently follows the active hovered month, while `aria-pressed` follows the selected month. That may be confusing for assistive tech because hover can temporarily change the announced selected option.
- The `role="listbox"` container is not itself focusable; keyboard control still works when a puck button is focused, but the `aria-activedescendant` pattern is incomplete unless the listbox itself receives focus.
- `build:demo` dirtied the tracked demo workbook because the generated XLSX is not deterministic or was out of date. I reverted that generated workbook after verification so the QA report/status are the only intended file changes.
