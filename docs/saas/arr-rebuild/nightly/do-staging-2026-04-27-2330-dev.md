## Summary

Implemented the highest-value small UX fix from the QA/Coordinator reports: the dashboard no longer presents final imported-period ARR as ambiguous “Latest ARR” / “Current ARR.” It now labels those numbers as ending/imported-period snapshots and adds an explicit recognition note explaining that ARR is period-based and defaults to the ending imported period, not automatically today’s calendar month.

This targets the main staging confusion QA found: upload/ingestion and API/dashboard correspondence were consistent, but the labels made final runoff-period ARR look like current operating ARR.
