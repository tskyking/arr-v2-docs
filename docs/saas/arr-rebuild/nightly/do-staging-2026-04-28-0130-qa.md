## Findings

### Green

1. Health, upload, import processing, review queue, dashboard API/UI correspondence, customer cube, and CSV exports all pass for the generated demo workbook.
2. Live staging data matches local expected data for all checked regression surfaces.
3. The 23:30 label/copy improvement is visible on live staging and materially reduces the previous “Latest ARR/current ARR” confusion.

### Still risky / blocked

1. **Persistence:** `/api/imports` was empty before this upload despite prior uploads. Imports should still be treated as ephemeral/resettable until durable storage or seeded demo behavior exists.
2. **Routing:** direct non-hash dashboard/customer/review URLs still 404 on staging. Use hash routes for demos, or configure SPA fallback.
3. **Product semantics:** labels are clearer, but Todd/Brian still need to decide the product default for headline ARR: current calendar month, latest invoice month, selected month, or ending imported period.
4. **End-date convention:** inclusive vs exclusive subscription end-date treatment still needs product/accounting decision and fixtures.
5. **Brian workbook adapter:** generated demo workbook works, but Brian-style `Contract Detail` workflow remains a separate customer-workflow gap.
6. **Browser interaction audit:** browser snapshot/screenshot worked, but click-driving was blocked by missing Playwright support in the gateway build.
