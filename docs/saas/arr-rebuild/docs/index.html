1. Introduction

ARR V2 is a tool for tracking and analyzing your company's Annual Recurring Revenue (ARR). It reads invoice and subscription data from your accounting system, calculates how much recurring revenue you have at any point in time, and helps you understand how that revenue is growing or changing.

Who is this for?

  • Finance and operations teams who need accurate ARR figures for board reporting, investor updates, or internal planning
  • Managers who want to understand which customers are expanding, contracting, or churning
  • Anyone responsible for reviewing and cleaning up subscription data before it flows into a revenue report

What it does not do (yet)

  • It does not connect directly to QuickBooks, Salesforce, or other systems — you export a file and upload it
  • It does not send invoices or manage billing

2. Getting Started

First Login

  1. Open the ARR V2 application in your browser.
  2. Sign in with the credentials provided by your administrator.
  3. You will land on the Dashboard page.

Navigation Overview

  • Dashboard — your ARR summary and trend charts
  • Import — upload a new data file
  • Review Queue — rows that need your attention before ARR is finalized
  • Movements — waterfall breakdown of ARR changes over time
💡 Tip: If you're using the tool for the first time, go to Import first. The dashboard and review queue won't show any data until you've uploaded a file.

3. Data Import

This is the most important section to get right. Everything downstream — the dashboard, the review queue, the movement analysis — depends on a clean import.

3.1 Supported File Formats

ARR V2 currently accepts Excel workbooks only (.xlsx format).

⚠️ Warning: .xls, .csv, .numbers, and other formats are not supported. Convert to .xlsx first.
⚠️ Warning: Password-protected workbooks cannot be read. Remove the password before uploading.

3.2 Required Workbook Structure

Your workbook must contain three specific sheets. The system detects them by content and column structure — sheet names are flexible.

Sheet 1: Transaction Detail

Your main invoice/transaction export from QuickBooks ("Sales by Customer Detail" format).

ColumnAccepted Names
Customer nameCustomer
Invoice dateDate or Invoice Date
Transaction typeTransaction Type
Invoice numberNum or Invoice Number
Product or serviceProduct/Service
QuantityQty
Sales priceSales Price
AmountAmount
💡 Tip: Subscription Start Date and End Date are critical for recurring items. Without them, the system flags rows and estimates a 1-year term from the invoice date.

Sheet 2: Product/Service Mapping

Maps each product/service name to a revenue category. If a product is not listed here, it will be flagged in the Review Queue.

⚠️ Product names must match exactly between the transaction sheet and this mapping sheet.

Sheet 3: Revenue Recognition Assumptions

Defines the rules for how each revenue category is recognized over time.

Rule TypeWhat it means
subscription_termSpread evenly over subscription start/end date range
fallback_one_year_from_invoiceAssume 1-year term if no subscription dates exist
fixed_36_months_from_invoiceSpread evenly over 36 months from invoice date
invoice_date_immediateRecognize full amount in invoice month (one-time fees)

3.3 Clean Data Checklist

  • File is .xlsx and not password-protected
  • Workbook has all three required sheets
  • Transaction rows have customer names, dates, products, and amounts
  • Recurring items have subscription start and end dates
  • Product/service mapping sheet covers all products in the transaction sheet
  • Recognition assumptions sheet has a rule for each product category

3.4 Step-by-Step Import

  1. From the main navigation, click Import.
  2. Click Choose File or drag and drop your .xlsx file.
  3. Click Upload.
  4. The system processes the file (usually a few seconds).
  5. On success, you'll see: rows processed, rows flagged, and date range covered.
  6. Click Go to Dashboard or Go to Review Queue.

3.5 Common Import Errors

ErrorCauseFix
Only .xlsx workbooks are supportedWrong formatExport as .xlsx
File could not be readEncrypted or damagedRemove password, re-export
Could not find a transaction detail sheetMissing sheet or wrong columnsAdd required columns (see 3.2)
Could not find product/service mapping sheetMissing sheetAdd mapping sheet

4. ARR Dashboard

The Dashboard shows your ARR at a glance — total ARR, trend over time, top customers, and a date range filter.

Understanding the ARR Number

ARR is the annualized value of active recurring subscriptions. It is not the same as cash collected or invoiced revenue in a given month.

  • A $12,000 annual subscription → $1,000/month ARR
  • A $1,200/month subscription → $14,400/year ARR
  • A one-time setup fee → $0 ARR

5. Customer Explorer

The Customer Explorer lets you browse ARR data at the individual customer level — useful for understanding a specific account's revenue history, peak ARR, or trend over time.

Customer List

Navigate to Customers from the main nav. You'll see all customers with ARR in the current import, sorted highest-first. Each row shows: customer name, current ARR, and ARR trend direction.

Customer Detail View

Click any customer to open their detail view:

  • ARR history — month-by-month chart in chronological order
  • Peak ARR — the highest ARR this customer has ever reached in your data
  • Current ARR — ARR in the most recent period
  • Active subscription lines — individual products/services driving ARR
💡 Tip: Peak ARR is a useful reference point during renewal conversations. A current ARR well below peak often signals contraction worth investigating.
⚠️ The customer list reflects the current import. Make sure you're viewing the right import for the period you're analyzing.

6. Review Queue

The Review Queue shows rows the system couldn't process with full confidence. Resolving these ensures your ARR numbers are accurate.

Flag Severity Levels

SeverityWhat it meansARR Impact
ErrorRow could not be processedExcluded from ARR until resolved
WarningProcessed using a system assumptionIncluded, but may be inaccurate
InfoAwareness onlyNo impact
⚠️ Warning: Error-severity rows are excluded from ARR. Unresolved errors will cause your ARR to be understated.

Common Flags

FlagSeverityWhat it means
MISSING_PRODUCT_SERVICE_MAPPINGErrorProduct not found in mapping sheet
MISSING_SUBSCRIPTION_DATES_FOR_RECURRING_ITEMWarningRecurring item missing start/end date — 1-year term assumed
SUSPICIOUS_NEGATIVE_AMOUNTWarningMay be a credit or refund
INVALID_DATEErrorDate couldn’t be parsed
AMOUNT_PRICE_QUANTITY_MISMATCHWarningAmount doesn’t match price × quantity
MISSING_INVOICE_NUMBERInfoInvoice number is blank
MISSING_RECOGNITION_ASSUMPTIONErrorNo recognition rule found for this category

Resolving a Flag

  1. Find the row in the Review Queue.
  2. Click to expand details.
  3. Choose Resolve (accept as-is) or Override (enter correct value).
⚠️ Important: Overrides are tied to the specific import they were applied to. If you re-import a corrected workbook, you must re-apply overrides — they do not carry forward automatically.

6. ARR Movement Analysis

The Movements page shows a waterfall breakdown of ARR changes period-over-period.

ComponentWhat it means
NewARR from customers with no prior ARR
ExpansionARR added from existing customers
ContractionARR lost from existing customers
ChurnARR lost from customers who left entirely

Formula: Closing ARR = Opening ARR + New + Expansion − Contraction − Churn

8. Troubleshooting

Dashboard shows no data

Fix: Go to Import and upload an .xlsx workbook.

ARR numbers look wrong

Fix: Check the Review Queue for unresolved flags. Verify recognition assumptions are correct.

Unexpected error on upload

Fix: Try again. If it persists, re-export from QuickBooks and try the fresh file.

10. Glossary

ARR — Annual Recurring Revenue. The annualized value of active recurring subscriptions.

Churn — When a customer cancels and their ARR drops to zero.

Expansion — An existing customer's subscription value increases.

Contraction — An existing customer's subscription value decreases.

Recognition Rule — How a revenue item's value is spread over time.

Review Item / Flag — A row the system couldn't process with full confidence.

Waterfall Chart — Shows how ARR moved period-to-period, broken down by New, Expansion, Contraction, and Churn.

Workbook — An Excel .xlsx file with the three required sheets.
