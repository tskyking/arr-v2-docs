# Source Material Review Pack - Summary for Brian

## Purpose
A first-pass public/reference pack has been assembled so you do not need to gather all source examples from scratch.

The goal is to give you a set of examples to react to, so we can refine the ingestion-tool plan more efficiently.

## What was assembled
A review pack was created with organized reference folders for:
- QuickBooks
- Xero
- NetSuite
- Chargebee
- Tabs market/product reference
- contract-example notes
- overall review notes

These materials are mostly public/vendor references and documentation summaries.

## Why this is useful
This pack helps with:
- identifying likely source field names
- seeing common export/import patterns
- comparing system conventions across platforms
- spotting likely differences between accounting/billing systems
- framing which source systems may be most practical for MVP support first

## Important limitation
These public/vendor examples are not a replacement for real-world enterprise edge cases.

They are most useful for:
- field-shape awareness
- format comparison
- source-system conventions
- initial ingestion design

They are less useful for:
- messy spreadsheet reality
- enterprise exceptions
- acquired-site rollups
- early-out clauses and nuanced contract structures
- custom finance-policy behavior

## Best use of your time
Rather than asking you to assemble all examples manually, the better next step is for you to review this pack and comment on it.

## Questions to answer while reviewing
1. Which examples resemble workflows you have actually seen?
2. Which important fields or structures are still missing?
3. Which edge cases matter most for the MVP?
4. Which source systems should be supported first?
5. What did your original tool do especially well that Tabs does not?

## Recommended outcome
If you annotate this pack at a high level, that should be enough to sharpen the ingestion design without requiring a large separate research effort from you.
