# Health & Family Project Memory

This folder is the durable working memory for health-family planning and follow-up.

## Purpose
- Preserve important decisions, notes, and context
- Avoid losing continuity across chats/sessions
- Keep open questions, action items, and stakeholder feedback in one place
- Provide a reliable fallback while semantic memory retrieval is unavailable

## Files
- `project-summary.md` — high-level current state
- `decision-log.md` — important decisions and why
- `feedback-log.md` — preferences, concerns, and stakeholder comments
- `open-questions.md` — unresolved items
- `backlog.md` — action list and next steps
- `session-notes.md` — chronological notes
- `sources-inventory.md` — relevant docs/files/data sources

## Working rule
Until memory retrieval is fixed, treat this folder as the source of truth for continuity.
